# ZeldaRecomp

Static recompilation of **Zelda LttP** (SNES) into native C, using the
[snesrecomp](https://github.com/mstan/snesrecomp) framework.

> **Scaffolded 2026-05-16 from the SMW template.** The
> framework (`snesrecomp/`) is shared with
> SuperMarioWorldRecomp and any other game projects — changes to it
> affect all sibling projects simultaneously. The per-game work below
> is what this repo is for.

## What "static recompilation" means here

The 65816 CPU code from the ROM is statically translated to C — every
function the game runs on the SNES's main CPU becomes a real
generated C function in `src/gen/`. **The rest of the SNES is not
recompiled** — PPU rendering, the APU/SPC700 audio coprocessor,
DMA/HDMA channels, hardware register I/O, and bank-mapping all run
through an embedded copy of snes9x's emulator core
(`snesrecomp/runner/snes9x-core/`). Same model as N64Recomp and
similar projects: recompile the CPU, emulate the silicon.

## Per-game files in this repo

| File | Purpose |
|---|---|
| `recomp/bank*.cfg` | Function declarations + hardware hints per ROM bank. The big iterative effort. |
| `recomp/funcs.h` | Auto-regenerated by `v2_regen.py`; never hand-edit. |
| `src/gen/` | Auto-regenerated C output; never hand-edit. `gitignored`. |
| `src/zelda_rtl.c` + `.h` | Host-side orchestration: PPU frame rendering, per-frame I_NMI/I_RESET/ProcessGameMode call order. Adapt from SMW's pattern. |
| `src/zelda_00.c` | Game-specific helper bodies (e.g. WRAM-init wrappers). Currently has just the orchestration stub. |
| `src/zelda_spc_player.c` + `.h` | SPC audio image loader. ROM→APU-RAM upload routine. |
| `src/zelda_cpu_infra.c` | Game-specific CPU infra (very thin). |
| `src/gen_stubs.c` | HLE bodies for code the recompiler can't synthesise (e.g. WRAM-executed). Empty at scaffold time. |
| `src/main.c` | SDL/audio/window glue. Copy of SMW's main with identifier renames; the body is largely game-agnostic but has a lot of SMW-era debug paths that may not apply. |
| `zelda.sln` + `src/zelda.vcxproj` | Visual Studio build glue. Auto-renamed from SMW's. The file list may need updating as new bank gen files appear. |
| `zelda.ini` | Auto-generated next to exe on first run. Sample copy at repo root. |

## Building

Prerequisites: Windows 10+, Visual Studio 2022, Python 3.9+ on PATH.

```bash
# 1. Drop the ROM at repo root (gitignored)
cp /path/to/your/legally-owned/zelda.sfc ./

# 2. Author at least the vectors in recomp/bank00.cfg (see TODOs there)

# 3. Initial regen
python snesrecomp/tools/v2_regen.py --rom zelda.sfc --cfg-dir recomp --out-dir src/gen

# 4. Build
"/c/Program Files/Microsoft Visual Studio/2022/Community/MSBuild/Current/Bin/MSBuild.exe" \
    zelda.sln -p:Configuration=Oracle -p:Platform=x64 -m -v:minimal
```

## snesrecomp linkage

`snesrecomp/` is a symlink to `../snesrecomp`, sharing one
checkout across all game projects. Framework changes are
instantly visible everywhere. Be careful committing framework
changes from this repo — they affect the SMW build and any other
game project simultaneously.
